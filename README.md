# My Portfolio API

Welcome to the My Portfolio API! This project is built using Node.js, Typescript, Express, Swagger for documentation, Prisma ORM, MySQL for the database, and Docker for containerization. With this API, you can manage and showcase your projects, blogs, tech stacks, and tags with Create, Read, Update, and Delete (CRUD) operations.

## Tech Stack

- Node.js
- Typescript
- Express
- Swagger (Documentation)
- Prisma ORM
- MySQL (database) -> Hosted on PlanetScale 
- Docker

## API Routes

### Projects

The projects route allows you to manage and showcase your projects. Each project has the following properties:

- `id`: Unique identifier
- `hero_image_link`: Link to the hero image
- `title`: Title of the project
- `sub_title`: Subtitle of the project
- `content`: Project description
- `date`: Date of the project
- `github_link`: Link to the project's GitHub repository
- `deployed_link`: Link to the deployed project
- `blog_link`: Link to the associated blog (if any)
- `type`: Type of project (e.g., personal, open source)
- `techstacks`: List of associated tech stacks
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp

### Tech Stacks

The tech stacks route allows you to manage and display the technologies used in your projects. Each tech stack has the following properties:

- `id`: Unique identifier
- `description`: Description of the tech stack
- `name`: Name of the tech stack
- `logo_link`: Link to the tech stack's logo
- `projects`: List of projects using this tech stack
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp

### Blogs

The blogs route enables you to manage your blog posts. Each blog has the following properties:

- `id`: Unique identifier
- `title`: Title of the blog
- `hero_image`: Link to the hero image
- `sub_title`: Subtitle of the blog (optional)
- `content`: Blog content
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp
- `tags`: List of associated tags (many-to-many relationship)

### Tags

The tags route allows you to manage tags for your blogs. Each tag has the following properties:

- `id`: Unique identifier
- `tag_name`: Unique name for the tag
- `tag_description`: Description of the tag (optional)
- `created_at`: Creation timestamp
- `updated_at`: Last update timestamp
- `blogs`: List of blogs associated with this tag (many-to-many relationship)

## Getting Started

To run this project, follow these steps:

1. Clone the repository.
2. Set up your MySQL database and update the database configuration in the project.
3. Install dependencies using `npm install`.
4. Run the API using the following scripts:
   - `npm run start:build` to compile TypeScript to JavaScript.
   - `npm run start:dev` to start the development server with TypeScript support.
   - `npm run start:prod` to compile TypeScript to JavaScript and start the production server.

5. Explore the API documentation generated by Swagger.

## API Documentation

The API is documented using Swagger, making it easy to understand and interact with the endpoints. You can access the API documentation at `http://your-api-url/docs`.

Please ensure that you have the required Node.js and TypeScript versions installed as specified in the `package.json` for compatibility with the project's dependencies.

Feel free to customize and enhance this project to showcase your portfolio in a unique way! Enjoy using the My Portfolio API.
